   <p>Вышел новый релиз PVS‑Studio&nbsp;— 7.23. В&nbsp;нём мы улучшили поддержку Qt Creator, увеличили стабильность работы PVS‑Studio Java, решили проблемы при&nbsp;анализе проектов, использующих NMake, добавили ряд новых диагностик и многое другое. Подробности в&nbsp;этой заметке.</p>

   <h3>Улучшения плагина для Qt Creator</h3>
   <p>В&nbsp;прошлом релизе мы выпустили плагин PVS‑Studio для&nbsp;Qt Creator. Выглядит он следующим образом (скриншот кликабельный):</p>
   <figure class="full-width ">
    <img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/7d6/3fa/fd1/7d63fafd19dcea1e43d5f65f942f7064.png" width="1213" height="795" data-src="https://habrastorage.org/getpro/habr/upload_files/7d6/3fa/fd1/7d63fafd19dcea1e43d5f65f942f7064.png">
    <figcaption></figcaption>
   </figure>
   <p>Примерно в&nbsp;тот&nbsp;же момент вышел Qt Creator 9. Конечно, поддержать новую версию уже тогда мы&nbsp;бы не&nbsp;успели. Однако теперь плагин работает и с&nbsp;Qt Creator версий 9.0.*.</p>
   <p>Более того, в&nbsp;релизе 7.23&nbsp;стало возможно использовать плагин для&nbsp;Qt Creator не&nbsp;только под&nbsp;Windows и Linux, но&nbsp;и под&nbsp;macOS на&nbsp;архитектуре x86–64.</p>
   <p>На&nbsp;данный момент плагин даёт возможность открывать отчёт, смотреть и фильтровать предупреждения, и&nbsp;т.&nbsp;п. В&nbsp;будущих релизах мы также планируем добавить в&nbsp;него возможность запуска анализа.</p>
   <p>Документация к&nbsp;плагину доступна по <a href="https://pvs-studio.com/ru/docs/manual/6648/">ссылке</a>.</p>
   <h3>Стабилизация Java-анализатора</h3>
   <p>Некоторое время назад мы обнаружили проблемы в&nbsp;работе нашего Java‑анализатора под&nbsp;системами семейства Unix. В&nbsp;ряде случаев анализатор сразу после запуска аварийно завершал работу и выдавал ошибку подобного вида:</p>
   <figure class="full-width ">
    <img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/392/854/7fc/3928547fc143a42dd565433c02da2c21.png" width="1387" height="421" data-src="https://habrastorage.org/getpro/habr/upload_files/392/854/7fc/3928547fc143a42dd565433c02da2c21.png">
    <figcaption></figcaption>
   </figure>
   <p>Ошибка&nbsp;была связана с&nbsp;проблемами загрузки динамических библиотек. В&nbsp;релизе 7.23&nbsp;она&nbsp;была устранена, и теперь Java разработчики могут спокойно использовать PVS‑Studio для&nbsp;анализа своих проектов под&nbsp;Unix системами:). Если&nbsp;же у&nbsp;вас всё ещё появляется эта или&nbsp;какая‑то другая ошибка, <a href="https://pvs-studio.com/ru/about-feedback/">напишите нам</a>, постараемся помочь.</p>

   <ul>
    <li><p><a href="https://pvs-studio.com/ru/docs/warnings/v1093/">V1093</a>. The result of the right shift operation will always be 0. The right operand is greater than or equal to the number of bits in the left operand.</p></li>
    <li><p><a href="https://pvs-studio.com/ru/docs/warnings/v1094/">V1094</a>. Conditional escape sequence in literal. Its representation is implementation-defined.</p></li>
   </ul>
   <p>C#</p>
   <ul>
    <li><p><a href="https://pvs-studio.com/ru/docs/warnings/v3184/">V3184</a>. The argument's value is greater than the size of the collection. Passing the value into the 'Foo' method will result in an exception.</p></li>
    <li><p><a href="https://pvs-studio.com/ru/docs/warnings/v3185/">V3185</a>. An argument containing a file path could be mixed up with another argument. The other function parameter expects a file path instead.</p></li>
    <li><p><a href="https://pvs-studio.com/ru/docs/warnings/v3186/">V3186</a>. The arguments violate the bounds of collection. Passing these values into the method will result in an exception.</p></li>
   </ul>
